Oppgave E5

det menes at nyere tagger som kommert i html 5 ikke nødvendigvis virker med eldre nettlesere,
men det gjør at nettleserne velger å foradre på ting så dem kan skjønne ting å vise det istedenfor å sitte fast.
dette gjør den ved å gå videre til tagger som den skjønner, å det er det som menes med fallback-løsningen,
som istedenfor å bruke en <.button> tag så kanskje du skal brukte <.input type="button"> istedenfor.
fallback i css er gjort på denne måten:
#cssNy {
opacity:0.5;
}
#cssGammel{
-ms-filter;"progid:DXImageTransform.Microsoft.Alpha(Opacity=50)";
filter:alpha(opacity=50);
}
#cssFallback{
-ms-filter;"progid:DXImageTransform.Microsoft.Alpha(Opacity=50)";
filter:alpha(opacity=50);
opacity:0.5;
}
denne koden viser da først den nye css egenskapen for gjennomsiktighet, den vill ikke virke i gamle nettelsere,
så for og forsikre seg bør amn da legge in den gamle css egenskapen for gjennomsiktighet i samme css sektion,
med det gamle overst. dette vill forsikre at nettelseren kan lese koden som den skal, men hvis den ikek skjønner
den siste funksjon så hopper den bare over den men det som står føre, altså den gamle versionen blir forsatt lest og virker.

oppgave E7

med boxmodel mener vi det at html er for det meste bygget opp i bokser med innhold,
grunn til at det er bra er at det grør det lettere og arbeide med det og lettere skiller opp i uliek elementer.
boxmodellen består av, margin, border, padding og content,

de ulike delerne av boxen blir styrt av css å gjør det urolig eltt å dele inn informajson og plasere det der du vill ah det.
margin er området rundt boxen som det ikke skal være noe annet, altså tomrum rindt boxen, men de kan kolapse å da går det feil.
border er kanten vi ser rudnt boxen, den kan styles i mange ulike variasjoner å viser hvor boxen er.
padding er tomrum mellom order og content,
content er det du putter inni boxen, det kan være hva som heldst.
